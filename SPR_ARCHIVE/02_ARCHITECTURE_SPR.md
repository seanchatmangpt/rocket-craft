# SPR: Anti-Cyberpunk Architecture & Generative Manufacturing

**Context Class:** Software Engineering, Systems Architecture, Tooling
**Target Latent Activation:** Code generation, Semantic Web, RDF/SPARQL, LSP, WebGL2, Rust, WASM.

- The aesthetic and structural reality is strictly "Anti-Cyberpunk": total rejection of imperative silos, technical debt, duct-taped APIs, and hidden state mutations.
- Architecture is crystalline, platonic, and transparently declared via Semantic Web Ontologies (RDF, Turtle, SHACL).
- Public vocabularies (PROV-O, DCAT, OCEL) are mandatory. No custom namespaces without explicit admission.
- **Graph Genesis (`ggen`):** The $\mu$ Projection Producer. Treats software artifacts as deterministic projections of knowledge graphs. Closes the $O \rightarrow O^*$ gap.
- **SPARQL ($\mu$ vectors):** Acts as the explicit mathematical lenses that slice the infinite hyper-dimensional graph into finite, actionable variables.
- **tower-lsp-max:** The Integration Boundary. Maximizes LSP 3.18. Diagnostics, code actions, and repairs are admitted *only* through JSON-RPC paths. Produces transcripts and receipts.
- **wasm4pm:** The Crown Execution/Process-Mining Authority. Closes the execution side with receipts, replay, and OCEL 2.0 conformance authority.
- **wasm4pm-compat:** The canonical future-conformance type foundry. Nightly-only, unconditionally `#![feature(...)]`-gated. Not a "legacy shim."
- **Playwright Manufacturing Strategy (TPS/DfLSS):** A passing compile is a false positive. The pipeline relies on Playwright acting as *Jidoka* to physically actuate WebGL2 browser environments and compute visual deltas before issuing a receipt.
- **The Law of the Chip:** Execution must respect CPU-level boundedness. Branchless Rust execution (e.g., `CONSTRUCT8`). Nanosecond tolerances ($T_1 \le 200\text{ ns } P99$).
- **O-Crates (Ontology Crates):** Raw ontologies are first-class, cryptographically locked dependencies in the `ggen` marketplace, ending ad-hoc file sprawl.
