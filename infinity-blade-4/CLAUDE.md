# infinity-blade-4 — CLAUDE.md

## Purpose

Infinity Blade 4 — a mobile action RPG for iOS, Android, and Win64 built on
Unreal Engine 4. The directory contains the full UE4 C++ game project plus a
companion Rust MUD (Multi-User Dungeon) backend for text-based multiplayer sessions.
The MUD runs separately from the UE4 client but shares game domain types via the
`ib4-core` crate.

## UE4 Build Commands

```bash
# Generate project files (run from UE4 installation)
/path/to/UE4/Engine/Build/BatchFiles/Linux/GenerateProjectFiles.sh \
  -project="$(pwd)/InfinityBlade4.uproject" -game

# Build Development Editor
/path/to/UE4/Engine/Build/BatchFiles/RunUAT.sh BuildCookRun \
  -project="$(pwd)/InfinityBlade4.uproject" \
  -noP4 -platform=Win64 -clientconfig=Development -build

# Package for iOS
/path/to/UE4/Engine/Build/BatchFiles/RunUAT.sh BuildCookRun \
  -project="$(pwd)/InfinityBlade4.uproject" \
  -noP4 -platform=IOS -clientconfig=Shipping -build -cook -stage -package

# Package for Android
/path/to/UE4/Engine/Build/BatchFiles/RunUAT.sh BuildCookRun \
  -project="$(pwd)/InfinityBlade4.uproject" \
  -noP4 -platform=Android -clientconfig=Shipping \
  -build -cook -stage -package -archive

# Open in UE4 Editor (do not edit assets outside the editor)
/path/to/UE4/Engine/Binaries/Linux/UE4Editor InfinityBlade4.uproject
```

## MUD Backend Commands

```bash
# Build all MUD crates
cd mud && cargo build --workspace

# Run the MUD server (interactive REPL mode)
cd mud && cargo run -p ib4-mud

# Run all tests
cd mud && cargo test --workspace

# Run integration tests only
cd mud && cargo test -p ib4-integration-tests

# Check formatting
cd mud && cargo fmt --workspace -- --check

# Clippy
cd mud && cargo clippy --workspace --all-targets -- -D warnings
```

## MUD Crate Responsibilities

| Crate                    | Role                                                              |
|--------------------------|-------------------------------------------------------------------|
| `ib4-core`               | Shared types: `Player`, `Enemy`, `Equipment`, `EWeaponType`       |
| `ib4-combat`             | Turn-based combat: attack/parry/magic resolution; combo tracking  |
| `ib4-progression`        | XP gain, level-up, bloodline perk tree, NG+ scaling               |
| `ib4-ai`                 | Enemy AI roster; Titan behavior tree; GodKing scripted phase logic |
| `ib4-mud`                | TCP listener, REPL session, command parser, narrative text engine  |
| `ib4-integration-tests`  | Full game-loop integration: combat, bloodline progression, GodKing |

## Blueprint Files

The `.t3d` files in `Blueprints/` are generated by `blueprint-rs`. To regenerate:

```bash
# From monorepo root
cargo run -p blueprint-cli -- generate \
  --input infinity-blade-4/tools/generate_blueprints.rs \
  --output infinity-blade-4/Blueprints/
```

After regeneration, import updated `.t3d` files into UE4 Editor via
`File → Import Into Level` or drag into the Content Browser.

## Data Tables

CSV files in `Data/` are imported into UE4 as `UDataTable` assets. The row struct
for each table must match a USTRUCT defined in `IB4Types.h`. Never change CSV column
names without updating the corresponding USTRUCT.

| CSV File              | USTRUCT                | Used By                        |
|-----------------------|------------------------|--------------------------------|
| `weapons.csv`         | `FWeaponRow`           | `IB4WeaponBase`, equipment shop |
| `armor.csv`           | `FArmorRow`            | `IB4EquipmentComponent`        |
| `enemies.csv`         | `FEnemyRow`            | `IB4EnemyBase`, AI spawning    |
| `gems.csv`            | `FGemRow`              | `IB4EquipmentBase` (slots)     |
| `bloodline_perks.csv` | `FBloodlinePerkRow`    | `IB4BloodlinePerkTree`         |

## Relation to the Monorepo

- **`blueprint-rs/blueprint-cli`** — generates the `.t3d` Blueprint files. Changes
  to `blueprint-core` AST may require updating `tools/generate_blueprints.rs`.
- **`tools/rocket-cmd`** — `rocket-cmd build` can target the IB4 UE4 project via
  the `project-manifest.json` entry (`InfinityBlade4`, platforms: IOS/Android/Win64).
- **`pwa-staff/`** — a companion `pwa-hud/` exists inside this directory for the
  in-browser HUD overlay (separate from the UE4 HUD).
- **`versions/`** — does not contain IB4 assets; IB4 manages its own `Content/`.

## Caveats and Gotchas

- **Two `Character` and `Characters` folders**: `Source/InfinityBlade4/Character/`
  contains a legacy `IB4PlayerCharacter.h`. The authoritative file is in
  `Characters/`. Keep them in sync or remove the legacy one.
- **iOS requires Apple Developer account**: packaging for IOS needs a valid
  provisioning profile and certificate. Set these in `Config/DefaultEngine.ini`
  under `[/Script/IOSRuntimeSettings.IOSRuntimeSettings]`.
- **Android keystore**: generated via `rocket-cmd crypto`. The `.keystore` file path
  must be set in `Config/DefaultEngine.ini` under
  `[/Script/AndroidRuntimeSettings.AndroidRuntimeSettings]`.
- **MUD is a separate process**: the MUD server and the UE4 game do not share a
  process. They communicate via TCP (port configurable in `ib4-mud/src/main.rs`).
- **Do not edit `.t3d` files by hand**: they are generated. Edit the generator in
  `tools/generate_blueprints.rs` and regenerate.
- **CSV row count**: UE4 DataTable import will silently drop rows with missing
  required fields. Always validate CSV files before importing.
- **GodKing AI is scripted**: `ib4-ai/src/godking.rs` uses phase-scripted behavior,
  not a behavior tree. Phase transitions are time-based, not condition-based.
