# Test Plan - E2E Testing for UE4 Universal RDF Mapping

## Goal
Establish a comprehensive, requirement-driven, and opaque-box test suite and testing infrastructure to validate all Turtle (.ttl) ontology files and SPARQL queries generated by the UE4 Universal RDF Mapping project.

## Methodology: 4-Tier Acceptance Testing
Our test cases are organized into four distinct tiers, covering features, boundaries, interactions, and realistic scenarios without implementation detail dependencies.

- **Tier 1: Feature Coverage (>= 5 cases per feature)**
  - Validates basic presence and basic definitions of key classes and properties.
  - Features:
    1. Core C++ Backbone (Classes like `UObject`, `AActor`, etc.)
    2. Subsystems (Physics, Rendering, Networking)
    3. Reflection & Blueprints (`UClass`, `UProperty`, execution nodes)
    4. Cooking & WASM Typestates

- **Tier 2: Boundary & Corner Cases (>= 5 cases per feature)**
  - Validates constraints, empty inputs, limits, naming conventions, and invalid types/properties.
  - Examples: Namespace checks, circular dependencies in subclass hierarchy, missing labels, invalid datatypes.

- **Tier 3: Cross-Feature Combinations (Pairwise Interaction Coverage)**
  - Checks how features interact (e.g. Blueprint nodes referencing C++ Core classes, Rendering systems invoking Reflection properties, Typestates compiling specific Actors).

- **Tier 4: Real-World Application Scenarios**
  - Exercises complete pathways: A complete mock character setup with physics rendering components, compiled/cooked to WASM.

## Test Infrastructure
1. **`ggen.toml`**: Configured to run validation, locate the Turtle files in `/Users/sac/.ggen/packs/ue4_ontology`, and enforce strictness settings.
2. **Validation Script / Harness**: A wrapper script that runs `ggen sync --validate-only` in the output pack directory and verifies the exit code.
3. **`TEST_INFRA.md`**: Detailing the test inventory, methodology, and mappings.
4. **`TEST_READY.md`**: Exposing test suite metrics and verifying that all tests are runnable and passing.
