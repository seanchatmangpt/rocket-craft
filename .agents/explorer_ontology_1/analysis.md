## 3. Mapping Strategy

Eden manufacturing concepts are systematically mapped to public standard ontologies to support standard query patterns, integration, and semantic reasoning.

### A. Assembly Component Hierarchy & Sockets (FIBO & SOSA)
The **Combinatorial Assembly Tree** represents the structural layout of manufactured assemblies.
- **`eden:AssemblyComponent`**: Declared as a subclass of `fibo:Asset` (since components represent inventory, products, and physical assets in the marketplace) and `fibo:Product` (since components are produced/manufactured items), as well as `sosa:FeatureOfInterest` (since the components are the target objects monitored by sensors/actuators).
- **`eden:Socket`**: Declared as a subclass of `sosa:Platform` since sockets represent connection interfaces that host components (e.g. hosting sensors, actuators, or subcomponents).
- Hierarchy and inverse properties like `eden:plugsInto` (domain: `eden:Part`, range: `eden:Socket`) and `eden:hasPart` (domain: `eden:Socket`, range: `eden:Part`) are formalized to represent structural linkages.

### B. Reliability Twin Properties (QUDT & SOSA)
The **Reliability Twin** monitors component health. All telemetry values are modeled using byte-class authority values (`xsd:unsignedByte`).
- **SOSA Integration**: Datatype properties (`eden:damageClass`, `eden:stressClass`, `eden:heatClass`, and `eden:fatigueClass`) are conceptually mapped to instances of `sosa:ObservableProperty` (e.g., `eden:damageProperty`, etc.) to define what physical properties the telemetry corresponds to.
- **QUDT Integration**: To ground these byte-class properties in physical dimensions, the datatype properties are annotated using the QUDT `qudt:hasQuantityKind` relation:
  - `eden:stressClass` $\rightarrow$ `qudt:Stress` (or `qudt:Pressure` kind).
  - `eden:heatClass` $\rightarrow$ `qudt:Temperature` kind.
  - `eden:damageClass` $\rightarrow$ `qudt:DimensionlessRatio` (normalized scale).
  - `eden:fatigueClass` $\rightarrow$ `qudt:DimensionlessRatio`.

### C. Delta Network Model (PROV-O)
The **Delta Network** replicates state transitions across object graphs.
- **`eden:Delta`**: Declared as a subclass of `prov:Entity`. In PROV-O, a `prov:Entity` is a physical, digital, or conceptual thing. Deltas are state-change payloads generated by activities.
- **`eden:ReceiptDelta`**: Declared as a subclass of `eden:Delta`. It models the visual validation receipt produced after browser actuation in the Unreal WASM environment. It is linked to the test/validation execution via `eden:receiptFor` (range: `prov:Activity`).
- **`eden:AuthorityDelta`**: Models authorization changes. Mapped to `prov:Agent` (for the authorizing identity) via `eden:authorizedBy` (subproperty of `prov:wasAttributedTo`).
- Properties like `eden:timestamp` are declared as subproperties of `prov:generatedAtTime` for native compatibility with standard provenance tools.

## 4. Draft Ontology Structure (`proposed_pack.ttl`)

Below is the verified RDF ontology structure designed for `ontology/pack.ttl`:

```turtle
@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .
@prefix eden: <https://ggen.io/ontology/eden-server/> .
@prefix fibo: <https://spec.edmcouncil.org/fibo/ontology/> .
@prefix sosa: <http://www.w3.org/ns/sosa/> .
@prefix qudt: <http://qudt.org/schema/qudt/> .
@prefix prov: <http://www.w3.org/ns/prov#> .

eden:Ontology a owl:Ontology ;
    owl:imports <http://www.w3.org/ns/prov.ttl> ,
                <https://raw.githubusercontent.com/w3c/sdw/gh-pages/ssn/integrated/sosa.ttl> ,
                <http://qudt.org/schema/qudt/> ,
                <https://spec.edmcouncil.org/fibo/ontology/> ;
    rdfs:label "Eden Manufacturing Server Ontology" ;
    rdfs:comment "Core ontology for the Eden Manufacturing Server. Integrates Industry 4.0 reliability engineering, sensor/fault loops, Combinatorial Assembly Trees, and byte-class state deltas with FIBO, SOSA, QUDT, and PROV-O." .

# =========================================================================
# SECTION 1: ASSEMBLY TREE TOPOLOGY (Mapped to FIBO & SOSA)
# =========================================================================

eden:AssemblyComponent a owl:Class ;
    rdfs:subClassOf fibo:Asset ,
                   fibo:Product ,
                   sosa:FeatureOfInterest ;
    rdfs:label "Assembly Component" ;
    rdfs:comment "A generic component within a Combinatorial Assembly Tree. Mapped to FIBO as an Asset/Product, and to SOSA as a Feature of Interest." .

eden:MechRoot a owl:Class ;
    rdfs:subClassOf eden:AssemblyComponent ;
    rdfs:label "Mech Root" ;
    rdfs:comment "The root structural component of a mechanical assembly (e.g. the chassis/frame of a vehicle or rig)." .

eden:SubAssembly a owl:Class ;
    rdfs:subClassOf eden:AssemblyComponent ;
    rdfs:label "Sub-Assembly" ;
    rdfs:comment "A sub-system of components grouped together under the root or another sub-assembly." .

eden:Part a owl:Class ;
    rdfs:subClassOf eden:AssemblyComponent ;
    rdfs:label "Part" ;
    rdfs:comment "An individual leaf component that plugs into a socket within the assembly." .

eden:Socket a owl:Class ;
    rdfs:subClassOf eden:AssemblyComponent ,
                   sosa:Platform ;
    rdfs:label "Socket" ;
    rdfs:comment "A connection point in the assembly where a Part can be plugged. Mapped to SOSA as a Platform hosting a Part as a hosted device." .

# --- Object Properties for Assembly Relations ---

eden:hasSocket a owl:ObjectProperty ;
    rdfs:domain eden:AssemblyComponent ;
    rdfs:range eden:Socket ;
    rdfs:label "has socket" ;
    rdfs:comment "Relates an assembly component (like a MechRoot or SubAssembly) to one of its connection sockets." .

eden:plugsInto a owl:ObjectProperty ;
    rdfs:domain eden:Part ;
    rdfs:range eden:Socket ;
    rdfs:label "plugs into" ;
    rdfs:comment "Relates a Part to the Socket in which it is connected." .

eden:hasPart a owl:ObjectProperty ;
    rdfs:domain eden:Socket ;
    rdfs:range eden:Part ;
    owl:inverseOf eden:plugsInto ;
    rdfs:label "has part" ;
    rdfs:comment "Relates a Socket to the Part plugged into it." .

eden:hasSubAssembly a owl:ObjectProperty ;
    rdfs:domain eden:AssemblyComponent ;
    rdfs:range eden:SubAssembly ;
    rdfs:label "has sub-assembly" ;
    rdfs:comment "Relates an assembly component to an nested sub-assembly." .

eden:isComponentOf a owl:ObjectProperty ;
    rdfs:domain eden:AssemblyComponent ;
    rdfs:range eden:AssemblyComponent ;
    rdfs:label "is component of" ;
    rdfs:comment "Relates a component to its parent component in the hierarchy." .

# =========================================================================
# SECTION 2: RELIABILITY TWIN PROPERTIES (Mapped to QUDT & SOSA)
# =========================================================================

# --- Observable Properties (SOSA Concept Mapping) ---

eden:damageProperty a sosa:ObservableProperty ;
    rdfs:label "Damage Property" ;
    rdfs:comment "The physical property of component degradation observed by telemetry." .

eden:stressProperty a sosa:ObservableProperty ;
    rdfs:label "Stress Property" ;
    rdfs:comment "The physical property of mechanical stress observed by telemetry." .

eden:heatProperty a sosa:ObservableProperty ;
    rdfs:label "Heat Property" ;
    rdfs:comment "The physical property of thermal state observed by telemetry." .

eden:fatigueProperty a sosa:ObservableProperty ;
    rdfs:label "Fatigue Property" ;
    rdfs:comment "The physical property of component structural fatigue observed by telemetry." .

# --- Datatype Properties with Byte-Class Authority Mappings (xsd:unsignedByte) ---

eden:damageClass a owl:DatatypeProperty ;
    rdfs:domain eden:AssemblyComponent ;
    rdfs:range xsd:unsignedByte ;
    qudt:hasQuantityKind qudt:DimensionlessRatio ;
    rdfs:seeAlso eden:damageProperty ;
    rdfs:label "damage class" ;
    rdfs:comment "The normalized damage metric of the component, represented as a byte-class authority value (0-255). Mapped to QUDT as a Dimensionless Ratio." .

eden:stressClass a owl:DatatypeProperty ;
    rdfs:domain eden:AssemblyComponent ;
    rdfs:range xsd:unsignedByte ;
    qudt:hasQuantityKind qudt:Stress ;
    rdfs:seeAlso eden:stressProperty ;
    rdfs:label "stress class" ;
    rdfs:comment "The mechanical stress metric of the component, represented as a byte-class authority value (0-255). Mapped to QUDT as a Stress / Pressure kind." .

eden:heatClass a owl:DatatypeProperty ;
    rdfs:domain eden:AssemblyComponent ;
    rdfs:range xsd:unsignedByte ;
    qudt:hasQuantityKind qudt:Temperature ;
    rdfs:seeAlso eden:heatProperty ;
    rdfs:label "heat class" ;
    rdfs:comment "The thermal status metric of the component, represented as a byte-class authority value (0-255). Mapped to QUDT as a Temperature kind." .

eden:fatigueClass a owl:DatatypeProperty ;
    rdfs:domain eden:AssemblyComponent ;
    rdfs:range xsd:unsignedByte ;
    qudt:hasQuantityKind qudt:DimensionlessRatio ;
    rdfs:seeAlso eden:fatigueProperty ;
    rdfs:label "fatigue class" ;
    rdfs:comment "The structural fatigue metric of the component, represented as a byte-class authority value (0-255). Mapped to QUDT as a Dimensionless Ratio." .

# --- SOSA Telemetry Integration ---

eden:ReliabilitySensor a owl:Class ;
    rdfs:subClassOf sosa:Sensor ;
    rdfs:label "Reliability Sensor" ;
    rdfs:comment "A telemetry sensor monitoring component health metrics (stress, heat, damage, fatigue)." .

# =========================================================================
# SECTION 3: DELTA NETWORK MODEL (Mapped to PROV-O)
# =========================================================================

eden:Delta a owl:Class ;
    rdfs:subClassOf prov:Entity ;
    rdfs:label "Delta" ;
    rdfs:comment "A digital state change payload or transaction in the replicated delta network. Mapped to PROV-O as a prov:Entity representing a state modification artifact." .

# --- The 5 Delta Families ---

eden:AuthorityDelta a owl:Class ;
    rdfs:subClassOf eden:Delta ;
    rdfs:label "Authority Delta" ;
    rdfs:comment "A delta recording governance, credential, or authority changes on the object graph. Attributed to an authorizing agent." .

eden:AssemblyDelta a owl:Class ;
    rdfs:subClassOf eden:Delta ;
    rdfs:label "Assembly Delta" ;
    rdfs:comment "A delta recording additions, removals, or connections in the Combinatorial Assembly Tree." .

eden:ProjectionDelta a owl:Class ;
    rdfs:subClassOf eden:Delta ;
    rdfs:label "Projection Delta" ;
    rdfs:comment "A delta capturing downstream projections, such as WASM/HTML5 packaging of the mechanical world." .

eden:InterestDelta a owl:Class ;
    rdfs:subClassOf eden:Delta ;
    rdfs:label "Interest Delta" ;
    rdfs:comment "A delta recording client or agent interest in specific sub-graphs of the assembly." .

eden:ReceiptDelta a owl:Class ;
    rdfs:subClassOf eden:Delta ;
    rdfs:label "Receipt Delta" ;
    rdfs:comment "A cryptographic receipt proving that a movement/actuation sequence was executed and visually validated in the WASM world. Mapped to PROV-O as a prov:Entity certifying a specific activity." .

# --- Delta Attributes and PROV-O Subproperty Mappings ---

eden:deltaId a owl:DatatypeProperty ;
    rdfs:domain eden:Delta ;
    rdfs:range xsd:string ;
    rdfs:label "delta ID" ;
    rdfs:comment "A unique cryptographic identifier or hash for the delta transaction." .

eden:sequenceNumber a owl:DatatypeProperty ;
    rdfs:domain eden:Delta ;
    rdfs:range xsd:unsignedLong ;
    rdfs:label "sequence number" ;
    rdfs:comment "Monotonically increasing sequence number for total ordering of deltas." .

eden:timestamp a owl:DatatypeProperty ;
    rdfs:subPropertyOf prov:generatedAtTime ;
    rdfs:domain eden:Delta ;
    rdfs:range xsd:dateTime ;
    rdfs:label "timestamp" ;
    rdfs:comment "The timestamp when the delta was produced. Mapped as a subproperty of prov:generatedAtTime." .

eden:targetGraph a owl:DatatypeProperty ;
    rdfs:domain eden:Delta ;
    rdfs:range xsd:anyURI ;
    rdfs:label "target graph" ;
    rdfs:comment "The URI of the target object graph or named graph to which this delta applies." .

eden:deltaPayload a owl:DatatypeProperty ;
    rdfs:domain eden:Delta ;
    rdfs:range xsd:string ;
    rdfs:label "delta payload" ;
    rdfs:comment "The serialized change payload, typically JSON, patch, or hex-encoded binary diff." .

# --- Delta Association Properties ---

eden:authorizedBy a owl:ObjectProperty ;
    rdfs:subPropertyOf prov:wasAttributedTo ;
    rdfs:domain eden:AuthorityDelta ;
    rdfs:range prov:Agent ;
    rdfs:label "authorized by" ;
    rdfs:comment "Links the AuthorityDelta to the prov:Agent that signed or authorized the state change. Subproperty of prov:wasAttributedTo." .

eden:appliesTo a owl:ObjectProperty ;
    rdfs:domain eden:Delta ;
    rdfs:range eden:AssemblyComponent ;
    rdfs:label "applies to" ;
    rdfs:comment "Links a Delta transaction to the AssemblyComponent whose state is modified." .

eden:receiptFor a owl:ObjectProperty ;
    rdfs:domain eden:ReceiptDelta ;
    rdfs:range prov:Activity ;
    rdfs:label "receipt for" ;
    rdfs:comment "Links the ReceiptDelta to the specific prov:Activity (such as visual validation and actuation in the WASM engine) it certifies." .
```
