## 2. Logic Chain

1. **Port mismatch and default configuration**:
   - The Playwright configuration expects the server to run on `http://localhost:3000` (based on its `baseURL` option).
   - The `start` script in `pwa-staff/package.json` invokes `local-web-server` without options. By default, `local-web-server` starts on port 8000.
   - Therefore, running `npm run start` currently does not satisfy Playwright's expectations, leading to `net::ERR_CONNECTION_REFUSED` unless manually configured.
   - Changing the start script to `"start": "local-web-server --port 3000"` directly resolves this port discrepancy.

2. **Web server lifecycle automation**:
   - Playwright requires the web server to be up and listening before tests can be executed.
   - Adding a `webServer` block inside `pwa-staff/playwright.config.ts` directs Playwright to automatically invoke the start script (`npm run start`), wait for port 3000 to become active (via the `url` option), and then start running tests.
   - Enabling `reuseExistingServer: !process.env.CI` allows faster local testing loops by reusing a running server instance if one already exists.

3. **Supabase Environment & Integration**:
   - The E2E auth test executes client-side authentication requests (`signUp`, `signInWithPassword`) which are sent to the Supabase local endpoint.
   - Since `supabaseClient.ts` uses fallback values:
     - URL: `http://127.0.0.1:54321` (matches local Kong gateway)
     - Anon key: `sb_publishable_ACJWlzQHlZjBrEguHvfOxg_3BJgxAaH` (matches the local publishable key generated by Supabase)
     and because the browser runs inside a sandbox that executes this bundled code, no manual environment variable injection is strictly required for local developer runs to pass.
   - However, to ensure that the environment variables are correctly managed, the testing environment could explicitly load `.env` variables if required.

4. **Missing Browser Executables**:
   - Since Playwright is configured to run tests across Chromium, Firefox, and Webkit projects, but only Chromium is locally installed, developers will encounter launch errors unless they install all browsers (`npx playwright install`) or restrict execution to Chromium.

## 3. Caveats

- We assume that the local Docker daemon is running and that the local Supabase container environment remains active. If Supabase is stopped, developers will need to run `npx supabase start` in the `supabase/` directory before running E2E tests.
- We did not investigate why the title in `index.html` differs from the assertion in `example.spec.ts` (`/PWA Staff/` vs `"Rocket Craft"`), resulting in a failure of `example.spec.ts`. This is outside the scope of the authentication flow task, but worth noting.

## 4. Conclusion

The E2E tests are configured to target `http://localhost:3000`, but there is no mechanism to launch or target the server on port 3000 automatically. By modifying the `package.json` start script and configuring `webServer` in `playwright.config.ts`, the E2E execution lifecycle can be fully automated.

### Proposed Code Changes

#### 1. Modify `pwa-staff/package.json`
Update the `"start"` script to run on port 3000.

**Before:**
```json
    "start": "local-web-server",
```

**After:**
```json
    "start": "local-web-server --port 3000",
```

#### 2. Modify `pwa-staff/playwright.config.ts`
Add the `webServer` block to automatically start the frontend server.

**Before:**
```typescript
export default defineConfig({
  testDir: './tests-e2e',
  fullyParallel: true,
  forbidOnly: !!process.env.CI,
  retries: process.env.CI ? 2 : 0,
  workers: process.env.CI ? 1 : undefined,
  reporter: 'html',
  use: {
    baseURL: 'http://localhost:3000',
    trace: 'on-first-retry',
  },
  projects: [
...
```

**After:**
```typescript
export default defineConfig({
  testDir: './tests-e2e',
  fullyParallel: true,
  forbidOnly: !!process.env.CI,
  retries: process.env.CI ? 2 : 0,
  workers: process.env.CI ? 1 : undefined,
  reporter: 'html',
  use: {
    baseURL: 'http://localhost:3000',
    trace: 'on-first-retry',
  },
  webServer: {
    command: 'npm run start',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
    stdout: 'ignore',
    stderr: 'pipe',
  },
  projects: [
```
