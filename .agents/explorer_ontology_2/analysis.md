# Detailed Design Strategy: Reliability Twins, Assembly Topology, and Delta Network Model

This report details the semantic architecture for the Eden Manufacturing Server's Combinatorial Assembly Trees, Reliability Twin properties, and the 5 Delta families. It provides the formal Turtle (`.ttl`) schema templates, rationale, and alignment strategies with industry standard ontologies (FIBO, SOSA, QUDT, and PROV-O).

---

## 1. Combinatorial Assembly Tree Topology Design

### 1.1 Overview
The Combinatorial Assembly Tree is a structural hierarchy representing the physical composition of a mechanical unit (e.g., a mech). To support dynamic, verifiable reconfigurations (mounting, unmounting, and replacing parts) while maintaining strict RDF validation, we employ the **Component-Socket Assembly Pattern**. 

Instead of a generic parent-child relation, the assembly tree alternates between **Components** and **Sockets**. This enables representing exact mounting positions (sockets) and enforces constraints on how parts connect.

### 1.2 Class Hierarchy
We define `eden:AssemblyComponent` as the abstract base class, which is subclassed as follows:
- `eden:MechRoot`: The foundational structural root of the mechanism (e.g., the primary chassis or core torso frame). There is exactly one per mechanism.
- `eden:SubAssembly`: A modular compound component grouping other sockets and parts (e.g., a forearm subassembly or a reactor core pod).
- `eden:Part`: A leaf component representing a single physical part that cannot contain further subassemblies or sockets (e.g., an armor plate, a laser lens, or a hydraulic actuator).
- `eden:Socket`: An explicit connection port or interface point on an assembly component. Components plug into sockets, and sockets belong to components.

### 1.3 Topology Properties
To construct the tree, we define four core object properties:
- `eden:hasSocket` (Domain: `MechRoot` or `SubAssembly`, Range: `Socket`): Specifies that a component has a connection port.
- `eden:socketOf` (Domain: `Socket`, Range: `MechRoot` or `SubAssembly`): The inverse of `hasSocket`.
- `eden:pluggedInto` (Domain: `Part` or `SubAssembly`, Range: `Socket`): Specifies that a component is plugged into a socket.
- `eden:hasPluggedComponent` (Domain: `Socket`, Range: `Part` or `SubAssembly`): The inverse of `pluggedInto`.

This pattern guarantees that parts do not plug directly into other parts without an intermediate socket, enabling precise tracking of physical mounting slots.

## 2. Reliability Twin Properties & Byte-Class Serialization

### 2.1 Edge Sync Rationale
High-frequency simulation and edge sync systems require low serialization overhead and high-performance validation. Decimal values, strings, and floats introduce floating-point precision mismatches and overhead in WebAssembly (WASM) runtimes.

To optimize, the Eden Manufacturing Server maps all reliability twin state metrics (damage, stress, heat, fatigue) to a discrete scale of **0 to 255**, represented in XML Schema as `xsd:unsignedByte`. This allows:
- **Direct 8-bit integer serialization** in network packets.
- **Bitmask operations** for state representation.
- **Deterministic state validation** (an out-of-range value is a structural protocol defect).

### 2.2 Property Definitions
We define four data properties on `eden:AssemblyComponent`:
- `eden:damageClass`: Component integrity status (0 = pristine, 255 = completely destroyed).
- `eden:stressClass`: Component structural stress (0 = idle, 255 = maximum yield limit).
- `eden:heatClass`: Thermal state (0 = absolute zero/ambient, 255 = thermal overload).
- `eden:fatigueClass`: Wear and tear cycles (0 = new, 255 = mechanical failure threshold).

## 3. Public Ontology Mappings (SOSA, QUDT, PROV-O, FIBO)

To ensure the Eden ontology is interoperable with standard Industry 4.0 systems, we align our classes and properties with established semantic standards:

| Eden Concept | Standard Class / Property | Namespace URI | Alignment Justification |
| :--- | :--- | :--- | :--- |
| `eden:AssemblyComponent` | `sosa:FeatureOfInterest` | `http://www.w3.org/ns/sosa/` | Components are physical systems whose properties (stress, damage, heat) are observed. |
| `eden:AssemblyComponent` | `prov:Entity` | `http://www.w3.org/ns/prov#` | The state of a component is a provenance entity that evolves via activities. |
| `eden:damageClass` (etc.) | `sosa:ObservableProperty` | `http://www.w3.org/ns/sosa/` | Metrics represent observable system properties subject to monitoring. |
| `xsd:unsignedByte` (value) | `qudt:Quantity` | `http://qudt.org/schema/qudt/` | Values are mapped to QUDT scale definitions to represent ratio metrics (e.g. 0-255 maps to 0.0-1.0 ratio). |
| `eden:Delta` | `prov:Entity` | `http://www.w3.org/ns/prov#` | Every state delta is a provenance-tracked entity generated by a system process. |
| `eden:AuthorityDelta` | `sosa:Actuation` | `http://www.w3.org/ns/sosa/` | Authority overrides represent actuator-driven mutations of the physical twin state. |
| `eden:ReceiptDelta` | `fibo-fnd-arr-doc:Document` | `https://spec.edmcouncil.org/fibo/ontology/` | Cryptographic receipts are formal audit documents proving compliance and verification. |

## 4. Draft Template: `ontology/pack.ttl`

Below is the complete, valid Turtle schema template for the core ontology. It imports standard vocabularies and defines the assembly tree and reliability twin structure.

```turtle
@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .
@prefix eden: <https://ggen.io/ontology/eden-server/> .
@prefix fibo: <https://spec.edmcouncil.org/fibo/ontology/> .
@prefix sosa: <http://www.w3.org/ns/sosa/> .
@prefix qudt: <http://qudt.org/schema/qudt/> .
@prefix prov: <http://www.w3.org/ns/prov#> .

eden:pack a owl:Ontology ;
    rdfs:label "Eden Manufacturing Server Core Ontology" ;
    rdfs:comment "Defines the core Combinatorial Assembly Tree topology and reliability twin properties for Eden Manufacturing Server." ;
    owl:imports <https://spec.edmcouncil.org/fibo/ontology/> ,
                <http://www.w3.org/ns/sosa/> ,
                <http://qudt.org/schema/qudt/> ,
                <http://www.w3.org/ns/prov#> .

# --- ASSEMBLY TREE CLASSES ---

eden:AssemblyComponent a owl:Class ;
    rdfs:label "Assembly Component" ;
    rdfs:comment "The base class for all mechanical parts, assemblies, and connections within the Combinatorial Assembly Tree." ;
    rdfs:subClassOf sosa:FeatureOfInterest , prov:Entity .

eden:MechRoot a owl:Class ;
    rdfs:label "Mechanical Root" ;
    rdfs:comment "The foundational node in the Combinatorial Assembly Tree. A mech has exactly one MechRoot from which all subassemblies and parts branch." ;
    rdfs:subClassOf eden:AssemblyComponent .

eden:SubAssembly a owl:Class ;
    rdfs:label "Sub-Assembly" ;
    rdfs:comment "A modular group of parts and sockets representing a functional subsystem (e.g., arm, powertrain)." ;
    rdfs:subClassOf eden:AssemblyComponent .

eden:Part a owl:Class ;
    rdfs:label "Part" ;
    rdfs:comment "A leaf node in the assembly tree representing an individual physical component (e.g., structural beam, actuator, armor plate)." ;
    rdfs:subClassOf eden:AssemblyComponent .

eden:Socket a owl:Class ;
    rdfs:label "Socket" ;
    rdfs:comment "A defined physical connection point on a MechRoot or SubAssembly. A socket may have at most one component plugged into it." ;
    rdfs:subClassOf eden:AssemblyComponent .

# --- ASSEMBLY PROPERTIES ---

eden:hasSocket a owl:ObjectProperty ;
    rdfs:label "has socket" ;
    rdfs:comment "Associates an assembly component (MechRoot or SubAssembly) with its interface sockets." ;
    rdfs:domain [ a owl:Class ; owl:unionOf (eden:MechRoot eden:SubAssembly) ] ;
    rdfs:range eden:Socket .

eden:socketOf a owl:ObjectProperty ;
    rdfs:label "socket of" ;
    rdfs:comment "Inverse of hasSocket; specifies which assembly component this socket belongs to." ;
    owl:inverseOf eden:hasSocket ;
    rdfs:domain eden:Socket ;
    rdfs:range [ a owl:Class ; owl:unionOf (eden:MechRoot eden:SubAssembly) ] .

eden:pluggedInto a owl:ObjectProperty ;
    rdfs:label "plugged into" ;
    rdfs:comment "Specifies that a Part or SubAssembly is physically mounted to a Socket." ;
    rdfs:domain [ a owl:Class ; owl:unionOf (eden:Part eden:SubAssembly) ] ;
    rdfs:range eden:Socket .

eden:hasPluggedComponent a owl:ObjectProperty ;
    rdfs:label "has plugged component" ;
    rdfs:comment "Inverse of pluggedInto; specifies which Part or SubAssembly is currently mounted to this Socket." ;
    owl:inverseOf eden:pluggedInto ;
    rdfs:domain eden:Socket ;
    rdfs:range [ a owl:Class ; owl:unionOf (eden:Part eden:SubAssembly) ] .

# --- RELIABILITY TWIN METRIC PROPERTIES ---

eden:damageClass a owl:DatatypeProperty ;
    rdfs:label "Damage Class" ;
    rdfs:comment "The integrity or damage level of the component represented as an unsigned byte value (0 to 255)." ;
    rdfs:domain eden:AssemblyComponent ;
    rdfs:range xsd:unsignedByte .

eden:stressClass a owl:DatatypeProperty ;
    rdfs:label "Stress Class" ;
    rdfs:comment "The current physical stress experienced by the component represented as an unsigned byte value (0 to 255)." ;
    rdfs:domain eden:AssemblyComponent ;
    rdfs:range xsd:unsignedByte .

eden:heatClass a owl:DatatypeProperty ;
    rdfs:label "Heat Class" ;
    rdfs:comment "The thermal status of the component represented as an unsigned byte value (0 to 255)." ;
    rdfs:domain eden:AssemblyComponent ;
    rdfs:range xsd:unsignedByte .

eden:fatigueClass a owl:DatatypeProperty ;
    rdfs:label "Fatigue Class" ;
    rdfs:comment "Cumulative structural fatigue or wear cycle count represented as an unsigned byte value (0 to 255)." ;
    rdfs:domain eden:AssemblyComponent ;
    rdfs:range xsd:unsignedByte .
```

## 5. Draft Template: `ontology/deltas.ttl`

Below is the complete, valid Turtle schema template for the delta ontology. It formalizes the 5 Delta families representing the full state synchronisation and verification lifecycle.

```turtle
@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .
@prefix eden: <https://ggen.io/ontology/eden-server/> .
@prefix fibo: <https://spec.edmcouncil.org/fibo/ontology/> .
@prefix sosa: <http://www.w3.org/ns/sosa/> .
@prefix qudt: <http://qudt.org/schema/qudt/> .
@prefix prov: <http://www.w3.org/ns/prov#> .

eden:deltas a owl:Ontology ;
    rdfs:label "Eden Manufacturing Server Delta Ontology" ;
    rdfs:comment "Defines the 5 Delta families for transaction streaming, interest management, projection, and verification." ;
    owl:imports <https://ggen.io/ontology/eden-server/> ,
                <https://spec.edmcouncil.org/fibo/ontology/> ,
                <http://www.w3.org/ns/sosa/> ,
                <http://qudt.org/schema/qudt/> ,
                <http://www.w3.org/ns/prov#> .

# --- BASE DELTA CLASS ---

eden:Delta a owl:Class ;
    rdfs:label "Delta" ;
    rdfs:comment "The abstract base entity representing any change transaction or event record in the server state space." ;
    rdfs:subClassOf prov:Entity .

# --- METADATA PROPERTIES FOR DELTAS ---

eden:sequenceNumber a owl:DatatypeProperty ;
    rdfs:label "Sequence Number" ;
    rdfs:comment "The monotonically increasing sequence identifier for the delta transaction." ;
    rdfs:domain eden:Delta ;
    rdfs:range xsd:nonNegativeInteger .

eden:timestamp a owl:DatatypeProperty ;
    rdfs:label "Timestamp" ;
    rdfs:comment "The UTC date and time when the delta record was committed." ;
    rdfs:domain eden:Delta ;
    rdfs:range xsd:dateTime .

# --- THE 5 DELTA FAMILIES ---

# 1. AuthorityDelta
eden:AuthorityDelta a owl:Class ;
    rdfs:label "Authority Delta" ;
    rdfs:comment "Represents state modifications forced by an external control authority (telemetry override)." ;
    rdfs:subClassOf eden:Delta , sosa:Actuation .

eden:targetComponent a owl:ObjectProperty ;
    rdfs:label "target component" ;
    rdfs:comment "The assembly component modified by the delta." ;
    rdfs:domain [ a owl:Class ; owl:unionOf (eden:AuthorityDelta eden:ProjectionDelta) ] ;
    rdfs:range eden:AssemblyComponent .

eden:targetProperty a owl:ObjectProperty ;
    rdfs:label "target property" ;
    rdfs:comment "The specific property on the component that was overridden." ;
    rdfs:domain eden:AuthorityDelta ;
    rdfs:range rdf:Property .

eden:oldValue a owl:DatatypeProperty ;
    rdfs:label "old value" ;
    rdfs:comment "The value of the property prior to the authority override." ;
    rdfs:domain eden:AuthorityDelta ;
    rdfs:range xsd:unsignedByte .

eden:newValue a owl:DatatypeProperty ;
    rdfs:label "new value" ;
    rdfs:comment "The value of the property after the authority override." ;
    rdfs:domain eden:AuthorityDelta ;
    rdfs:range xsd:unsignedByte .

eden:authorityToken a owl:DatatypeProperty ;
    rdfs:label "authority token" ;
    rdfs:comment "Cryptographic authorization token verifying the authority's mandate." ;
    rdfs:domain eden:AuthorityDelta ;
    rdfs:range xsd:string .

# 2. AssemblyDelta
eden:AssemblyDelta a owl:Class ;
    rdfs:label "Assembly Delta" ;
    rdfs:comment "Represents structural changes in the Combinatorial Assembly Tree (mounting, unmounting, or replacing)." ;
    rdfs:subClassOf eden:Delta .

eden:affectedComponent a owl:ObjectProperty ;
    rdfs:label "affected component" ;
    rdfs:comment "The component that is being mounted, unmounted, or replaced." ;
    rdfs:domain eden:AssemblyDelta ;
    rdfs:range eden:AssemblyComponent .

eden:affectedSocket a owl:ObjectProperty ;
    rdfs:label "affected socket" ;
    rdfs:comment "The socket involved in this assembly modification." ;
    rdfs:domain eden:AssemblyDelta ;
    rdfs:range eden:Socket .

eden:assemblyAction a owl:DatatypeProperty ;
    rdfs:label "assembly action" ;
    rdfs:comment "The structural action performed. Allowed values are 'MOUNT', 'UNMOUNT', or 'REPLACE'." ;
    rdfs:domain eden:AssemblyDelta ;
    rdfs:range xsd:string .

eden:componentPlugged a owl:ObjectProperty ;
    rdfs:label "component plugged" ;
    rdfs:comment "The component plugged into the socket (if any) during a MOUNT or REPLACE action." ;
    rdfs:domain eden:AssemblyDelta ;
    rdfs:range eden:AssemblyComponent .

# 3. ProjectionDelta
eden:ProjectionDelta a owl:Class ;
    rdfs:label "Projection Delta" ;
    rdfs:comment "Represents the translation of reliability twin state into game engine parameters or physical simulation weights." ;
    rdfs:subClassOf eden:Delta .

eden:projectionTarget a owl:DatatypeProperty ;
    rdfs:label "projection target" ;
    rdfs:comment "The identifier of the engine target property or variable (e.g., render visibility, physics mass scale)." ;
    rdfs:domain eden:ProjectionDelta ;
    rdfs:range xsd:string .

eden:projectionValue a owl:DatatypeProperty ;
    rdfs:label "projection value" ;
    rdfs:comment "The actual value mapped into the engine runtime context." ;
    rdfs:domain eden:ProjectionDelta ;
    rdfs:range xsd:string .

eden:projectionScale a owl:DatatypeProperty ;
    rdfs:label "projection scale" ;
    rdfs:comment "The scale factor applied during physical mapping." ;
    rdfs:domain eden:ProjectionDelta ;
    rdfs:range xsd:decimal .

# 4. InterestDelta
eden:InterestDelta a owl:Class ;
    rdfs:label "Interest Delta" ;
    rdfs:comment "Represents changes in subscriber data-sync filters and visibility boundaries." ;
    rdfs:subClassOf eden:Delta .

eden:subscriberId a owl:DatatypeProperty ;
    rdfs:label "subscriber ID" ;
    rdfs:comment "Unique client or agent session identifier." ;
    rdfs:domain eden:InterestDelta ;
    rdfs:range xsd:string .

eden:interestTarget a owl:ObjectProperty ;
    rdfs:label "interest target" ;
    rdfs:comment "The assembly component acting as the root of the subscribed sub-tree." ;
    rdfs:domain eden:InterestDelta ;
    rdfs:range eden:AssemblyComponent .

eden:interestAction a owl:DatatypeProperty ;
    rdfs:label "interest action" ;
    rdfs:comment "The interest filter operation. Allowed values are 'SUBSCRIBE' or 'UNSUBSCRIBE'." ;
    rdfs:domain eden:InterestDelta ;
    rdfs:range xsd:string .

eden:depthLimit a owl:DatatypeProperty ;
    rdfs:label "depth limit" ;
    rdfs:comment "The maximum depth limit of tree traversal to synchronize from the interest target." ;
    rdfs:domain eden:InterestDelta ;
    rdfs:range xsd:nonNegativeInteger .

# 5. ReceiptDelta
eden:ReceiptDelta a owl:Class ;
    rdfs:label "Receipt Delta" ;
    rdfs:comment "Represents the cryptographic verification, visual audit trail, and validation verdict for a set of transactions." ;
    rdfs:subClassOf eden:Delta .

eden:receiptHash a owl:DatatypeProperty ;
    rdfs:label "receipt hash" ;
    rdfs:comment "The SHA-256 cryptographic verification digest of the delta record or transaction bundle." ;
    rdfs:domain eden:ReceiptDelta ;
    rdfs:range xsd:string .

eden:relatedDelta a owl:ObjectProperty ;
    rdfs:label "related delta" ;
    rdfs:comment "The delta or transaction record that this receipt validates." ;
    rdfs:domain eden:ReceiptDelta ;
    rdfs:range eden:Delta .

eden:visualDelta a owl:DatatypeProperty ;
    rdfs:label "visual delta" ;
    rdfs:comment "The quantified difference metric computed by Playwright visual test (e.g. motion difference percentage)." ;
    rdfs:domain eden:ReceiptDelta ;
    rdfs:range xsd:decimal .

eden:screenshotUri a owl:DatatypeProperty ;
    rdfs:label "screenshot URI" ;
    rdfs:comment "A link/URI pointing to the visual snapshot artifact captured during verification." ;
    rdfs:domain eden:ReceiptDelta ;
    rdfs:range xsd:anyURI .

eden:auditVerdict a owl:DatatypeProperty ;
    rdfs:label "audit verdict" ;
    rdfs:comment "The formal validation status ('PASS' or 'FAIL') based on Playwright visual confirmation rules." ;
    rdfs:domain eden:ReceiptDelta ;
    rdfs:range xsd:string .

eden:receiptSignature a owl:DatatypeProperty ;
    rdfs:label "receipt signature" ;
    rdfs:comment "Cryptographic signature certifying the verification run." ;
    rdfs:domain eden:ReceiptDelta ;
    rdfs:range xsd:string .
```
