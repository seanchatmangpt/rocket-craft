# SPR: COMBINATORIAL MAXIMALISM FOR AAA UE4 MECH ASSETS

## Core Doctrine
Stop incremental improvement of single renders. Construct the full AAA mech asset possibility space, manufacture candidates across that space, refuse everything below production gates, and admit only game-ready UE4 assets.

AAA_UE4_MECH = Geometry × Materials × Textures × Rig × Sockets × Collision × LOD × Animation × GameplayBytes × UE4Import × Receipts - RefusedForms

## Parallel Maximalism Swarms
- **Swarm A (Geometry)**: Generate 100-1,000 body candidates. Refuse primitive blobs.
- **Swarm B (Hard-surface)**: Generate armor shells, bevels, panel lines, greebles.
- **Swarm C (Material/Texture)**: Generate PBR families (ceramic armor, dark frame, emissive, wear).
- **Swarm D (Rig/Socket)**: Generate skeleton, IK targets, weapon sockets, hit zones.
- **Swarm E (UE4 Import/Cook)**: Export FBX, bind materials, cook, headless screenshot.
- **Swarm F (IP/Admissibility)**: Run external expressive-cluster distance gates.

## Admission Gates
- **GEO (Geometry)**: No duplicate part files. Valid mesh topology. Part graph matches declared skeleton. LOD0/1/2 generated. Collision hulls generated.
- **MOR (Morphology)**: Component graph similarity. Armor segmentation density. Tapered/overlapping wing morphology. Core head/torso ratios. False-positive blobs refused.
- **MAT (Material)**: PBR channels exist. Material IDs match UE4 import table.
- **RIG (Rig)**: Skeleton, joint hierarchy, sockets, damage zones, IK targets exist.
- **UE4 (Engine)**: FBX export succeeds. UE4 command-line import, skeletal mesh creation, material/texture binds, cook, and headless screenshot all succeed. Asset loads in test map.
- **EVD (Evidence)**: OCEL log, receipt chain, replay success, LSP diagnostics match verifier report.
