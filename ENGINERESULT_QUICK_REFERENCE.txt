================================================================================
EngineResult Typestate Pattern — QUICK REFERENCE
================================================================================

FOUR STATES (Zero-Sized Markers):
  1. ScanComplete   — raw observations collected
  2. Enriched       — observations classified by file type/category
  3. Evaluated      — diagnostics generated with blocking flags
  4. Deduped        — duplicates removed, final state (no transitions)

STATE TRANSITIONS (Only these are legal):
  ScanComplete —(enrich)—> Enriched —(evaluate)—> Evaluated —(dedupe)—> Deduped
                                                         └—(emit)—> Vec[Diagnostic]

================================================================================
USAGE: BEFORE vs AFTER
================================================================================

BEFORE (Unsafe — Manual Dedup):
  let obs = scan_file("x.rs");
  let diags = evaluate_diagnostics(&obs);

  // Manual dedup (easy to forget!)
  let mut seen = HashSet::new();
  let unique = diags.into_iter()
    .filter(|d| seen.insert((d.file_path.clone(), d.line, d.code.clone())))
    .collect();

AFTER (Safe — Type-Enforced):
  let result = EngineResult::from_file_scan("x.rs")
    .enrich()
    .evaluate(&config)
    .dedupe();

  let unique = result.emit();  // Guaranteed unique!

================================================================================
IMPOSSIBLE OPERATIONS (Compile-Time Errors)
================================================================================

❌ .emit_blocking() before .dedupe()
   EngineResult<Evaluated> has no emit_blocking() method

❌ .enrich() twice
   EngineResult<Enriched> has no enrich() method

❌ .evaluate() before .enrich()
   EngineResult<ScanComplete> has no evaluate() method

❌ .dedupe() before .evaluate()
   EngineResult<Enriched> has no dedupe() method

================================================================================
VALID PATTERNS
================================================================================

✅ Full pipeline with dedup:
   EngineResult::from_file_scan(path)
     .enrich()
     .evaluate(&config)
     .dedupe()
     .emit()

✅ Partial pipeline (pre-dedup):
   EngineResult::from_file_scan(path)
     .enrich()
     .evaluate(&config)
     .emit()  // OK: EngineResult<Evaluated> allows emit()

✅ Filtered queries on pre-dedup state:
   let result = EngineResult::from_file_scan(path)
     .enrich()
     .evaluate(&config);
   result.filter_by_blocking(true)

✅ Multiple queries on deduped state:
   let result = EngineResult::from_file_scan(path)
     .enrich()
     .evaluate(&config)
     .dedupe();
   let all = result.emit();
   let blocking = result.emit_blocking();
