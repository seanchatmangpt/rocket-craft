# unify-ffi Gap Report: Node.js N-API Bindings

## Status

The `napi` feature (`--features napi`) adds real N-API bindings via `napi-rs 2.x`.
Without it, the crate is a pure Rust library usable from Rust code but NOT loadable
as a Node.js native module.

## What was added (G9 fix)

| File | Change |
|------|--------|
| `Cargo.toml` | Added optional `napi`, `napi-derive` (deps) and `napi-build` (build-dep); added `[features] napi` |
| `build.rs` | Emits `cargo:rustc-cfg=napi` when `CARGO_FEATURE_NAPI` is set |
| `src/napi_bindings.rs` | `#[napi]`-annotated `dispatch`, `version`, `list_commands` (behind `#[cfg(feature = "napi")]`); always-available `dispatch_raw`, `version_raw`, `list_commands_raw` helpers |
| `src/lib.rs` | `pub mod napi_bindings`; re-exports `dispatch_raw`, `version_raw`, `list_commands_raw` |
| `tests/ffi_dispatch.rs` | Integration tests for all raw helpers (run without napi feature) |

## To build the `.node` binary

### Prerequisites

```bash
# Install napi-rs CLI
npm install -g @napi-rs/cli

# Ensure Rust nightly or stable with the right targets
rustup target add x86_64-unknown-linux-gnu   # or your host triple
```

### Build command (napi-rs CLI — preferred, handles renaming)

```bash
cd unify-rs/unify-ffi
napi build --features napi --release
# Outputs: unify_ffi.node in the current directory
```

### Build command (plain cargo)

```bash
cd unify-rs
cargo build -p unify-ffi --features napi --release
# Output: target/release/libunify_ffi.so  (Linux)
#         target/release/libunify_ffi.dylib (macOS)
# Rename to unify_ffi.node before require()-ing from Node.
```

### Node.js usage after build

```javascript
const { dispatch, version, listCommands } = require('./unify_ffi.node');

console.log(version());          // "0.1.0"
console.log(listCommands());     // ["echo", "ping", "version"]
console.log(dispatch('ping', 'null'));   // '"pong"'
console.log(dispatch('echo', '"hi"'));   // '"hi"'
```

## Remaining work

1. **`package.json` wrapper** — scaffold an npm package in `unify-rs/unify-ffi/` with
   `@napi-rs/cli` build scripts so that `npm run build` produces `unify_ffi.node` and
   `index.js` / TypeScript typings are auto-generated by `napi-rs`.

2. **GitHub Actions matrix** — add a workflow job that cross-compiles `.node` binaries
   for all target triples and uploads them as release artifacts.

3. **npm publish** — publish as `@rocket-craft/unify-ffi` once the matrix is green.

4. **Wire real MCP registry** — `unify_dispatch_ffi` currently calls
   `FfiCommandRegistry::new().with_builtins()` (only `ping`, `echo`, `version`).
   It should be wired to the full `unify-mcp` tool registry so that all MCP tools
   are reachable from Node.js over FFI.

5. **TypeScript type generation** — `napi-rs` CLI generates `.d.ts` declarations
   automatically; commit them alongside the `.node` binary or in the npm package.

6. **Error code surface** — `FfiError::code` is currently swallowed into the napi
   `GenericFailure` status. Consider mapping known codes (`NOT_FOUND`, `PARSE_ERROR`,
   `INVALID_ARG`) to distinct `napi::Status` values or including the code in the
   JS Error `message` field for easier client-side handling.
